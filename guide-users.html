rhelpi18n

Reading R Help in Your Language

How to set up your R environment so that ?function shows help pages in Spanish, Chinese, Portuguese — whatever translation is available.

Who this is for You use R and would rather read documentation in your own language. Nothing here changes how your code works — only what the help pages look like.

The whole thing in 4 lines

remotes::install_github("adit-0132/rhelpi18n@dynamic-sexpr")  # 1. the engine
remotes::install_github("SOMEONE/theirpkg.es")                # 2. a translation
# 3. RESTART R
Sys.setenv(LANGUAGE = "es"); library(rhelpi18n); ?hello       # 4. read it

The rest of this page explains each step and what to do when something doesn't work.

1 Install the engine

rhelpi18n is what makes R look for a translation whenever you open a help page. You install it once.

install.packages("remotes")
remotes::install_github("adit-0132/rhelpi18n@dynamic-sexpr")
Note on the branch Until this work is merged upstream, install from the @dynamic-sexpr branch. Afterwards it will simply be remotes::install_github("eliocamp/rhelpi18n").

2 Install the package you want help for

Nothing special — install it however you normally would.

install.packages("theirpkg")                       # from CRAN
# or
remotes::install_github("someone/theirpkg")        # from GitHub

3 Install a translation

Translations ship as their own small packages, usually named <package>.<language>. Install one the same way:

remotes::install_github("SOMEONE/theirpkg.es")     # Spanish translation

You can install as many as you like — one per language, or for several different packages. They don't conflict.

Where do translations come from? They're written and published by translators (often the community). If none exists for your package and language yet, you'll simply keep seeing English — nothing breaks.

4 Choose your language

R uses an environment variable called LANGUAGE.

For this session only

Sys.setenv(LANGUAGE = "es")

Permanently (recommended)

Add one line to your ~/.Renviron file so it applies every time you start R:

# open the file (creates it if needed)
usethis::edit_r_environ()

# add this line, save, and restart R:
LANGUAGE=es

No usethis? Just create/edit ~/.Renviron in any text editor.

LanguageSet LANGUAGE to
Spanishes
Frenchfr
Brazilian Portuguesept_BR
Simplified Chinesezh_CN
Traditional Chinesezh_TW
Back to Englishen
Underscores, not hyphens Write zh_CN, not zh-CN. And be specific where it matters: zh_CN (Simplified) and zh_TW (Traditional) are treated as different languages — a bare zh may pick either one.

5 Restart R, then read the help

# Session → Restart R   (or Ctrl+Shift+F10 in RStudio)

library(rhelpi18n)
library(theirpkg)
?hello

The help pane now shows the translated page. Anything that's version- or date-specific still shows the real, current value — translations don't freeze those.

Sections without a translation stay in English If a translator has only done part of a page, you'll see translated sections mixed with English ones. That's intentional — you always get correct information, never a stale one.

6 Switching languages

Sys.setenv(LANGUAGE = "fr"); ?hello   # French
Sys.setenv(LANGUAGE = "en"); ?hello   # original English

You don't need to reinstall anything to switch — just change the variable. (If a page seems stuck, restart R.)

Troubleshooting

SymptomWhat to check
Help is still in English Did you restart R after installing? Then check Sys.getenv("LANGUAGE") returns your code, and that library(rhelpi18n) is loaded.
Some sections translated, some not Normal — that translation is incomplete, or the package was updated since it was written.
Whole page reverted to English after updating the package The translation was written for the older version. Check for an updated translation, or let the translator know.
Nothing happens at all Confirm the translation module is installed: "theirpkg.es" %in% rownames(installed.packages())

Check what translations you have

ip <- installed.packages(fields = c("Translates", "Language"))
ip[!is.na(ip[, "Translates"]), c("Package", "Translates", "Language")]

This lists every translation module installed, which package it translates, and its language.

How it works (the short version)

When you open a help page, rhelpi18n:

  1. notices which package the page belongs to and what LANGUAGE you've set;
  2. looks through your installed packages for a translation module declaring that package and language;
  3. swaps the translated text in — while re-inserting any live values (dates, versions, generated tables) from your actual installation.

That last point is the important one: a translation stays correct even after you reinstall or update the package, because the changing parts are filled in fresh each time.
